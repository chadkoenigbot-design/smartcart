# SmartCart UI/UX Audit — 2026-03-31

## Scope

Full review of the SmartCart static web app (HTML/CSS/JS, GitHub Pages) covering accessibility, responsiveness, semantic HTML, and UI polish across all six sections: Home, Profile, Planner, Optimizer, Budget, and How It Works.

---

## Findings

### Accessibility

| Issue | Severity | Status |
|-------|----------|--------|
| **No visible keyboard focus indicators** — all interactive elements used `outline: none` with no `:focus-visible` replacement. Keyboard-only users had zero visual feedback. | High | Fixed |
| **Missing `role="tabpanel"` on tab panels** — nav buttons had `role="tab"` and `aria-selected`, but the corresponding panels lacked `role="tabpanel"` and `aria-labelledby`, breaking the ARIA tabs pattern for screen readers. | High | Fixed |
| **No skip-to-content link** — keyboard users had to tab through the entire header and sidebar before reaching page content. | Medium | Fixed |
| **Multiple `<main>` elements** — five separate `<main>` tags existed (one per tab panel), violating the HTML spec requirement of a single `<main>` landmark per page. Screen readers could not identify the primary content region. | Medium | Fixed |
| Quantity stepper buttons use `tabindex="-1"` — acceptable for mouse interaction, but the qty input itself has no `aria-label`. | Low | Noted |
| Health/cuisine checkboxes are wrapped in `<label>` elements (good), but the checkbox groups lack `<fieldset>`/`<legend>` grouping. | Low | Noted |

### Responsiveness

| Finding | Notes |
|---------|-------|
| Solid mobile-first approach with breakpoints at 480, 540, 600, 640, 767, and 768px. | Well-structured |

## Implemented Improvements (5)

### 1. Global `:focus-visible` keyboard focus ring
- **Files:** `style.css`
- Replaced scattered `outline: none` with a global `:focus { outline: none }` / `:focus-visible` pattern.
- All interactive elements now show a 2px green focus ring when navigated via keyboard, with no ring on mouse click.
- Nav tabs get an inset offset to fit within their rounded containers.

### 2. ARIA tabpanel roles and labelling
- **Files:** `index.html`
- Added `role="tabpanel"` and `aria-labelledby` (referencing both header and sidebar tab button IDs) to all 6 active tab panels.
- Added missing `id` attributes to header nav-tab buttons for `goals`, `optimizer`, `budget`, and `how` so `aria-labelledby` can reference them.

### 3. Single `<main>` landmark
- **Files:** `index.html`
- Replaced five inner `<main class="main">` elements (one per tab) with `<div class="main">`.
- Promoted the `app-content` wrapper to `<main id="main-content">` — the single landmark for the entire page.

### 4. Skip-to-content link
- **Files:** `index.html`, `style.css`
- Added a visually hidden `<a class="skip-link">` that appears on keyboard focus, linking to `#main-content`.
- Lets keyboard users bypass header and sidebar navigation in a single Tab press.

### 5. How It Works page copy and markup
- **Files:** `index.html`
- Removed "Placeholder" from the hero badge.
- Rewrote hero description, card headings/descriptions, and status section with concrete, confident copy.
- Added `feature-card-title` and `feature-card-desc` classes for visual consistency with Home page cards.

## Recommendations for Future Work

1. **Add `<fieldset>` + `<legend>` around checkbox/radio groups** (health concerns, cuisines, budget style, shop frequency) for better screen reader context.
2. **Add `aria-label` to the quantity input** (`#qty-input`) — e.g., "Item quantity".
3. **Implement arrow-key navigation within the tablist** per the ARIA tabs pattern (Left/Right to move between tabs, Home/End for first/last).
4. **Add `aria-live="polite"` to dynamic result areas** (`#results-content`, `#goal-results`, `#budget-summary-content`) so screen readers announce updates.
5. **Consider a print stylesheet** — users may want to print their shopping strategy or budget summary.
6. **Add `<meta name="description">` and Open Graph tags** for better link previews when shared.
7. **Lazy-load Unsplash hero images** with `loading="lazy"` on an `<img>` element (current `background-image` approach can't be lazy-loaded natively).
8. **Reduce CSS specificity wars** — a few `!important` declarations on sidebar nav could be refactored.
